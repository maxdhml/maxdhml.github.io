## Ce que j'ai fait

J'ai conçu et déployé des applications web en maîtrisant l'architecture client-serveur, de la structure visuelle jusqu'à la logique métier et la sécurité.

* **Développement Front-End (R1.09 / SAÉ 1.04) :** J'ai réalisé mon portfolio personnel (maxdhml.github.io) en utilisant HTML5 pour la structure et CSS3 pour le design. J'ai mis en place un Responsive Design adaptatif (mobile, tablette, bureau) en m'appuyant sur le framework Bootstrap.
* **Développement Back-End et Sécurité (R2.09) :** J'ai appris à créer des sites dynamiques avec le framework Python Flask. J'ai également été sensibilisé aux vulnérabilités web, notamment les failles XSS et les injections SQL, et j'ai appris à sécuriser les formulaires et les bases de données.
* **Normes et Déploiement :** J'ai veillé au respect des standards du W3C, utilisé des outils d'inspection de navigateur pour déboguer le code et géré le déploiement via GitHub Pages en utilisant des chemins relatifs pour garantir la portabilité des fichiers.

## Pourquoi

Le Web est l'interface principale de la plupart des services réseau aujourd'hui. Comprendre son architecture permet non seulement de créer des outils de présentation, mais aussi de comprendre comment les données circulent via le protocole HTTP entre un client et un serveur. La maîtrise du Back-end est cruciale pour automatiser des services, tandis que la sensibilisation à la cybersécurité est indispensable pour protéger les infrastructures contre les attaques courantes qui exploitent les entrées utilisateurs.

## Comment

J'ai utilisé Visual Studio Code pour l'écriture du code. Pour la partie visuelle, j'ai adapté un template Bootstrap en utilisant des classes utilitaires pour la mise en page. Pour la partie serveur (R2.09), j'ai utilisé Python et Flask pour gérer les routes. J'ai systématiquement vérifié la validité de mon code via le validateur du W3C et inspecté les requêtes HTTP et l'arborescence du DOM dans la console développeur de mon navigateur pour optimiser le rendu et le chargement des ressources.

## Mes difficultés

**La gestion de la priorité CSS et du Responsive.** Lors de la personnalisation de mon portfolio, ma principale difficulté a été de faire cohabiter mes propres styles CSS avec les classes préexistantes de Bootstrap. Il m'est arrivé que des modifications visuelles ne s'appliquent pas à cause de la spécificité des sélecteurs. De plus, intégrer un CV complet en HTML/CSS (et non en simple PDF) tout en conservant une lisibilité parfaite sur smartphone a nécessité une gymnastique importante sur les points de rupture (breakpoints) des grilles Bootstrap.

## Ce que j'en ai appris

J'ai compris que le Web est un écosystème de couches superposées. J'ai appris à différencier le rôle du navigateur (exécution du code client) de celui du serveur (traitement des données et logique). J'ai également réalisé l'importance de la séparation des préoccupations : le contenu (HTML), le style (CSS) et la logique (Python/Flask). Enfin, j'ai pris conscience que la sécurité web n'est pas optionnelle : le filtrage des entrées est un réflexe vital pour éviter qu'un pirate ne détourne une base de données.

## Ce que je ferais autrement

Pour mes prochains projets web, je passerai davantage de temps sur la phase de maquettage avant de coder. De plus, au lieu de modifier directement les fichiers CSS, j'utiliserais des préprocesseurs pour rendre le style plus modulaire. Enfin, j'approfondirais l'utilisation de Docker (évoqué en TP bonus) pour "conteneuriser" mes applications web, garantissant ainsi que mon environnement de développement soit identique à mon environnement de production, ce qui facilite grandement le déploiement sur des serveurs distants.
